What are Parameter IDs??

Parameter IDs(PIDs) are codes used to request data from a vehicle, used as a diagnostic tool. SAE standard J/1979 defines many PIDs, but manufacturers also define many more PIDs specific to their vehicles. Typically, an automotive technician will use PIDs with a scan tool connected to the vehicle's OBD-II connector. PIDS include: Status of the OBD II System Component Monitors (Ready or Complete, or Not Ready or Incomplete) Live Sensor Data (Oxygen sensor rich/lean indication, coolant temperature, MAP value, TPS value, vehicle speed, mass air flow, ambient temperature, engine rpm, etc.) Status of Switches or Devices (cruise control on/off. brake pedal switch on/off. TCC engaged/disengaged, etc.) Long and Short Term Fuel Trim, O2 sensor cross counts, injector duration.


Task at hand

We will be sending all the encoded data collected from the OBD to the Raspberry Pie. Then the Raspberry Pie will decode the current data and send it to the Firebase Database using the inbuilt wi-fi function present in Raspberry Pie. The Firebase Database will get the Values and print the values on the Webpage because of its built-in Realtime functionality function. Also we will be pointing the location of the cars whose data are stored in the Firebase.
